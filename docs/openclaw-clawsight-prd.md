# OpenClaw `clawlens` 产品文档

## 背景

Claude Code 的 `/insights` 命令通过分析用户过去 30 天的会话历史，生成使用报告，帮助用户发现使用模式、摩擦点和改进方向，获得了广泛好评。

OpenClaw 作为全能型个人 AI 助手，具备持久记忆、多渠道通信、自主行为、可扩展 Skills 等特性，与 Claude Code 的"终端编程工具"定位有本质区别。`clawlens`（claw + insight）在借鉴 Claude Code 基础分析框架的同时，围绕"助手"而非"工具"的定位，增加了记忆健康度、自主行为审计、多渠道分析等独有维度。

输出为 Markdown 格式纯文本，用户在任意聊天渠道内即可直接阅读，无需打开外部页面。

---

## 报告维度

### 1. 使用概览

基础统计，让用户快速了解使用规模。

- 会话数、消息数、活跃天数
- Token 消耗量及趋势
- 活跃时间分布（时段、星期）

### 2. 任务分类

将所有交互按意图归类，展示时间和精力的分配。

- 分类示例：邮件管理、日程协调、信息检索、内容创作、编码辅助、自动化脚本、智能家居控制、文件操作等
- 各类别占比及趋势变化
- 标记高价值任务（节省时间最多、重复频率最高）

### 3. 摩擦分析

识别协作障碍，区分责任归属。

- **Claw 侧**：误解指令、执行错误、输出不符预期、未遵循用户偏好
- **用户侧**：上下文不足、指令模糊、在不合适的渠道发起复杂任务
- 列出 Top 摩擦模式及具体示例

### 4. 多渠道分析

分析用户在不同通信渠道上的使用模式。

- 各渠道（WhatsApp / Telegram / Discord / Slack / iMessage / 终端等）的使用量占比
- 各渠道的任务类型分布
- 渠道错配检测及优化建议

### 5. 自主行为审计

汇报 Claw 在 heartbeat、cron job、后台任务等场景下的自主行为表现。

- 自主行为总次数及类别
- 用户确认/采纳率 vs 否决/忽略率
- 标记异常或越界行为
- 准确度趋势（随时间是否在改善）

### 6. Skills 生态分析

分析 Skills 的安装和使用情况。

- 已安装 vs 实际使用的 Skills 数量对比
- 高频 Skills 排名
- 未使用 Skills 清理建议
- Claw 自建 Skills 的使用率评估
- 基于任务模式推荐 ClawHub 社区 Skills

### 7. 改进建议

基于以上维度，输出可直接执行的建议。

- **Quick Wins**：立即可做的改进（补充关键记忆、启用未用 Skill 等）
- **功能推荐**：用户尚未使用的 OpenClaw 功能
- **委托建议**：基于当前委托深度，建议下一步可以交给 Claw 的任务类型

---

## 实现阶段

| 阶段 | 内容 | 说明 |
|------|------|------|
| P0 | 使用概览 + 任务分类 + 摩擦分析 + Skills 生态分析 | 数据易获取，用户感知直接 |
| P1 | 自主行为审计 | OpenClaw 核心差异化维度 |
| P2 | 多渠道分析 + 委托建议 | 需要更多数据积累 |
